Copyframe 隐私政策

生效日期：2026 年 7 月 22 日

Copyframe 仅在用户明确点击“复制整页”“选择元素”或“导出离线网页”后，将当前网页转换为可粘贴到 Figma 的设计图层或下载的离线网页包。

我们处理的数据

在用户主动发起一次采集时，Copyframe 会在浏览器本机读取当前网页中与导出有关的内容：页面结构、可见文本、布局和样式信息、图片和字体等资源引用，以及已填写的非密码表单内容。密码输入框的值不会被采集。

“复制到 Figma”会把生成的 HTML/SVG 内容写入系统剪贴板；Copyframe 不会代替用户上传内容到 Figma 或其他服务。

为实现“重新复制上次采集结果”，扩展会在 Chrome 本地存储中保留最近一次采集的页面标题、网址、图标、诊断信息和生成的粘贴内容。这些记录最多保留 24 小时，用户可在扩展的“上次采集”中点击“清除本机记录”立即删除。

“导出离线网页”会在本机生成 ZIP 文件。生成过程中的临时 ZIP 数据在交给浏览器下载后删除；最终下载文件保留在用户选择的位置，由用户自行管理和删除。

数据传输与第三方

Copyframe 不运营接收采集内容的服务器，也不会将页面内容、剪贴板内容、浏览历史、分析数据或个人信息出售、出租、共享或传输给开发者或第三方。

为了将当前页面的图片、字体、样式等资源嵌入导出结果，扩展可能直接向该页面引用的资源地址发起浏览器请求。请求从用户的浏览器直接发送到原网页或其资源提供方，不会经过 Copyframe 服务器；在用户已经登录相关网站时，请求可能携带该网站正常使用所需的浏览器凭据。用户可在首次采集时拒绝网站访问权限；此时不会开始采集。

用户主动点击“获取 Viewer”时，扩展会打开 Copyframe Viewer 的 GitHub 发布页。该链接不会自动打开，也不会向 Copyframe 发送采集内容。

权限说明

  • activeTab 与 scripting：仅在用户点击扩展后读取并处理当前网页。
  • 可选网站访问权限：在用户开始采集时请求，用于读取网页引用的跨域图片、字体和样式资源；不会在未操作时执行。
  • clipboardWrite：按用户操作把生成的设计内容写入剪贴板。
  • storage 与 unlimitedStorage：临时保存最近一次采集结果和生成下载文件所需的数据。
  • downloads 与 offscreen：在本机生成并下载离线网页 ZIP。

用户选择

Copyframe 不会在后台持续监测网页。用户可以在浏览器扩展管理页撤销网站访问权限、卸载扩展，或使用扩展内的“清除本机记录”删除最近一次采集内容。

联系方式

如对本隐私政策或数据处理有疑问，请通过 Copyframe Viewer GitHub Issues 联系我们。
